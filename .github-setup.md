# GitHub Repository Setup Guide

This guide will help you push the aDOs IDE project to GitHub.

## Quick Setup (Automated)

If you have GitHub CLI installed and authenticated:

```powershell
.\setup-github.ps1
```

## Manual Setup

### Option 1: Using GitHub CLI (Recommended)

1. **Authenticate with GitHub CLI** (if not already):
   ```powershell
   gh auth login
   ```

2. **Create repository and push**:
   ```powershell
   gh repo create ados-ide --public --source=. --remote=origin --description "A local, AI-augmented IDE and mini-OS for cognitive development" --push
   ```

### Option 2: Using GitHub Web Interface

1. **Create a new repository on GitHub**:
   - Go to: https://github.com/new
   - Repository name: `ados-ide`
   - Description: `A local, AI-augmented IDE and mini-OS for cognitive development`
   - Visibility: Public (or Private, your choice)
   - **Do NOT** initialize with README, .gitignore, or license (we already have these)

2. **Add remote and push**:
   ```powershell
   git remote add origin https://github.com/Jakedoes1111/ados-ide.git
   git branch -M main
   git push -u origin main
   ```

### Option 3: Using SSH (if you have SSH keys set up)

```powershell
git remote add origin git@github.com:Jakedoes1111/ados-ide.git
git branch -M main
git push -u origin main
```

## Verify

After pushing, verify the repository:

```powershell
git remote -v
```

You should see:
```
origin  https://github.com/Jakedoes1111/ados-ide.git (fetch)
origin  https://github.com/Jakedoes1111/ados-ide.git (push)
```

Visit your repository at: https://github.com/Jakedoes1111/ados-ide

## Troubleshooting

### Authentication Issues

If you get authentication errors:
- For HTTPS: Use a Personal Access Token (PAT) instead of password
- For SSH: Ensure your SSH key is added to GitHub
- For GitHub CLI: Run `gh auth login` and follow the prompts

### Large Files

If you encounter issues with large files:
- The repository includes build artifacts that are gitignored
- If needed, you can use Git LFS for large binary files

### Branch Name

If your default branch is not `main`:
```powershell
git branch -M main
```
